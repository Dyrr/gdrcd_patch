13 ## 2 Requisiti Minimi {#mainpage_2} #
14 I requisiti minimi per poter utilizzare questa versione del GDRCD 7 sono:
15 - GDRCD 5.5 (testato) o versioni inferiori (da testare)
16 - Uno spazio web su un host che PHP 7.1 o superiore
17 - Un database con MySQL 5.1 ao superiore (MySQL 5.6 consigliato);
18 - Le seguenti estensioni di php abilitate :
19  + mysqli o mysqlind a scelta (PDO consigliata);
20  + openssl;
21  + gdlib.

25 ### 3.1 Installazione su GDRCD nuovo {#mainpage_3_1} #
26 Per prima cosa bisogna estrarre i file dall'archivio compresso .zip scaricato dal repository o dal sito contenente il
27 pacchetto in una cartella del vostro pc controllando che la struttura delle cartelle dello stesso sia stata mantenuta.
28 Una volta fatta questa verifica i file vanno caricati sullo spazio web che avete scelto. Per effettuare questa
29 operazione avrete bisogno di un client FTP come programma per caricare i file sull'host e dei dati di accesso all'ftp
30 del vostro spazio web.
31 Per il client ftp ci sono numerosi ottimi programmi freeware che fanno allo scopo, scegliete quello che vi attira di più.
32 Per i dati di accesso all'ftp, solitamente l'host li invia nella mail di iscrizione o comunque ha delle guide al riguardo
33 per cui vi rimando alla documentaizone dell'host.
34 Una volta configurato il client ftp, basta caricare i file sull'host che è la stessa cosa di quando copiate dei file da
35 una cartella all'altra del vostro pc e verificare per scrupolo che la sturttura delle directory sia rimasta la stessa e
36 la patch sarà stata installata automaticamente e non dovrete fare altro.

38 ### 3.1 Installazione su GDRCD nuovo {#mainpage_3_1} #
39 Nel caso si voglia installare la patch su un GDRCD su cui si è già lavorato la procedura è molto simile alla precedente.
40 Gli unici controlli che vanno fatti sono nei file **includes/required.php** e nel file **header.inc.php**.
41 - Se non avete fatto modifiche a questi file si può semplicemente procedere all'installazione come al punto
42 [Installazione su GDRCD nuovo](#mainpage_3_1).
43 - Se sono state fatte delle modifiche a quei file, cancellate la versione di quei due file presente nella patch e poi
44 aprite la vostra versione dei file e fate le seguenti modifiche:
45 
46 **header.inc.php**
47 ```php
48 <?php
49  //SE SI È IL GESTORE E
50  //SE LA PAGINA RICHIESTA È QUELLA DI GESTIONE
51  if( $_SESSION['permessi'] == SUPERUSER
52  && $_REQUEST['page'] == 'gestione') {
53  //aggiorna il file del css
54 ?>
55  <link rel="stylesheet"
56  href="<?php echo csscrush_file('themes/' . $PARAMETERS['themes']['current_theme'] . '/css/source/gdrcd.css'); ?>"
57  type="text/css" />
58 <?php
59  //IN CASO CONTRARIO
60  } else {
61  //carica il css preprocessato
62 ?>
63  <link rel="stylesheet" href="themes/<?php echo $PARAMETERS['themes']['current_theme']; ?>/gdrcd.css" type="text/css" />
64 <?php
65  }
66 ?>
67  <link rel="stylesheet" href="https://use.fontawesome.com/releases/v5.15.3/css/all.css" crossorigin="anonymous">
68 ```
69 Questro blocco di codice va aggiunto esattamente prima del tag <title>.
70 
71 **includes/required.php**
72 ```php
73  //include il file con la definizione della root
74  require_once(__DIR__ . '/../root.inc.php');
75 
76  //carica il set di funzioni per l'interfacciarsi con il database
77  \functions\load('core/db/mysqli');
78 
79  //Eseguo la connessione al database
80  //per questioni di retrocompatibilità ho lasciato anche la vechcia connesiuone al database.
81  $db = \gdrcd\db\connect(
82  $PARAMETERS['database']['username'],
83  $PARAMETERS['database']['password'],
84  $PARAMETERS['database']['url'],
85  $PARAMETERS['database']['database_name'],
86  $PARAMETERS['database']['collation']
87  );
88 
89  //include il preprocessore dei css
90  require_once ROOT . '/system/lib/csscrush/CssCrush.php';
91 
92  //opzioni per il processore dei css
93  $settings= array(
94  'minify' => true,
95  'output_dir' => ROOT . '/themes/' . $PARAMETERS['themes']['current_theme'],
96  'versioning' => true,
97  'formatter' => 'block'
98  );
99  //imposta le opzioni per il processore dei css
100  csscrush_set('options',$settings);
101 ```
102 Questo blocco di codice va aggiunto alla fine del codice prima del tag di chiusura di php.

104 ## 4 Funzionalità introdotte {#mainpage_4} #
105 La patch introduce diversi strumenti rispetto alla vecchia versione del GDRCD, che seppur non visibili subito ad occhio
106 nudo forniscono delle utilità molto comode per lo sviluppo modulare del pacchetto anche in vista di una nuova versione
107 o lo sviluppo di moduli nuovi.
108 La patch contiene:
109 - Una integrazione con il preprocessore di css [csscrush](https://the-echoplex.net/csscrush/) pre avere una gestione dei
110 css più organizzata. Attualmente i css del gdrcd non sono stati modificati ma si può trovare nella cartella
111 **themes/advanced/css/source/gdrcd.css** e nei vari file presenti in quella sezione i file per iniziare a lavorare con il
112 preprocessore.
113 - Un nuovo gestore delle connessioni e operazioni con il database
114 - Dei set di funzioni per il caricamento delle risorse (funzioni,monuli,template) per facilizzare la gestione del lavoro.
115 I vari set di sunzioni hann l aloro guida specifica nelle varie pagine di questa documentaizone.
