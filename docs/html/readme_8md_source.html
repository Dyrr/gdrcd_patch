13 ## 2 Requisiti Minimi {#mainpage_2} #
14 I requisiti minimi per poter utilizzare questa versione del GDRCD 7 sono:
15 - GDRCD 5.5 (testato) o versioni inferiori (da testare)
16 - Uno spazio web su un host che PHP 7.1 o superiore
17 - Un database con MySQL 5.1 ao superiore (MySQL 5.6 consigliato);
18 - Le seguenti estensioni di php abilitate :
19  + mysqli o mysqlind a scelta (PDO consigliata);
20  + openssl;
21  + gdlib.

23 ## 3 Installazione {#mainpage_3} #
24 
25 ### 3.1 Installazione su GDRCD nuovo {#mainpage_3_1} #
26 Per prima cosa bisogna estrarre i file dall'archivio compresso .zip scaricato dal repository o dal sito contenente il
27 pacchetto in una cartella del vostro pc controllando che la struttura delle cartelle dello stesso sia stata mantenuta.
28 Una volta fatta questa verifica i file vanno caricati sullo spazio web che avete scelto. Per effettuare questa
29 operazione avrete bisogno di un client FTP come programma per caricare i file sull'host e dei dati di accesso all'ftp
30 del vostro spazio web.
31 Per il client ftp ci sono numerosi ottimi programmi freeware che fanno allo scopo, scegliete quello che vi attira di più.
32 Per i dati di accesso all'ftp, solitamente l'host li invia nella mail di iscrizione o comunque ha delle guide al riguardo
33 per cui vi rimando alla documentaizone dell'host.
34 Una volta configurato il client ftp, basta caricare i file sull'host che è la stessa cosa di quando copiate dei file da
35 una cartella all'altra del vostro pc e verificare per scrupolo che la sturttura delle directory sia rimasta la stessa e
36 la patch sarà stata installata. l'unica modifica da fare sarà questa:
37 
38 **config.inc.php**
39 ```php
40 $PARAMETERS['database']['collation'] = 'utf8mb4'; //set di caratteri del db
41 ```
42 Questo blocco di codice va aggiunto appena sotto gli altri tre parametri di connesisone al database.

44 ### 3.1 Installazione su GDRCD preesistente {#mainpage_3_1} #
45 Nel caso si voglia installare la patch su un GDRCD su cui si è già lavorato la procedura è molto simile alla precedente.
46 Gli unici controlli che vanno fatti sono nei file **includes/required.php** e nel file **header.inc.php** e **ref_header.inc.php**.
47 - Se non avete fatto modifiche a questi file si può semplicemente procedere all'installazione come al punto
48 [Installazione su GDRCD nuovo](#mainpage_3_1).
49 - Se sono state fatte delle modifiche a quei file, cancellate la versione di quei tre file presenti nella patch e poi
50 aprite la vostra versione dei file e fate le seguenti modifiche:
51 
52 **config.inc.php**
53 ```php
54 $PARAMETERS['database']['collation'] = 'utf8mb4'; //set di caratteri del db
55 ```
56 Questo blocco di codice va aggiunto appena sotto gli altri tre parametri di connesisone al database.
57 
58 **header.inc.php**
59 ```php
60 <?php
61  //SE SI È IL GESTORE E
62  //SE LA PAGINA RICHIESTA È QUELLA DI GESTIONE
63  if( $_SESSION['permessi'] == SUPERUSER
64  && $_REQUEST['page'] == 'gestione') {
65  //aggiorna il file del css
66 ?>
67  <link rel="stylesheet"
68  href="<?php echo csscrush_file('themes/' . $PARAMETERS['themes']['current_theme'] . '/css/source/gdrcd.css'); ?>"
69  type="text/css" />
70 <?php
71  //IN CASO CONTRARIO
72  } else {
73  //carica il css preprocessato
74 ?>
75  <link rel="stylesheet" href="themes/<?php echo $PARAMETERS['themes']['current_theme']; ?>/gdrcd.css" type="text/css" />
76 <?php
77  }
78 ?>
79  <link rel="stylesheet" href="https://use.fontawesome.com/releases/v5.15.3/css/all.css" crossorigin="anonymous">
80 ```
81 Questro blocco di codice va aggiunto esattamente prima del tag <title>.
82 
83 **ref_header.inc.php**
84 ```php
85  <link rel="stylesheet" href="../themes/<?php echo $PARAMETERS['themes']['current_theme']; ?>/gdrcd.css" type="text/css" />
86  <link rel="stylesheet" href="https://use.fontawesome.com/releases/v5.15.3/css/all.css" crossorigin="anonymous">
87 ```
88 Questro blocco di codice va aggiunto esattamente prima del tag <title>.

90 **includes/required.php**
91 ```php
92  //include il file con la definizione della root
93  require_once(__DIR__ . '/../root.inc.php');
94  //include il file con il set di funzioni per le risorse
95  require_once ROOT . '/system/inc/functions/core/resources.inc.php';
96 
97  //carica il set di funzioni per l'interfacciarsi con il database
98  \functions\load('core/db/mysqli');
99  //carica gli helpers
100  \functions\load('core/helpers');
101 
102  //Eseguo la connessione al database
103  //per questioni di retrocompatibilità ho lasciato anche la vechcia connesiuone al database.
104  $db = \gdrcd\db\connect(
105  $PARAMETERS['database']['username'],
106  $PARAMETERS['database']['password'],
107  $PARAMETERS['database']['url'],
108  $PARAMETERS['database']['database_name'],
109  $PARAMETERS['database']['collation']
110  );
111 
112  //include il preprocessore dei css
113  require_once ROOT . '/system/lib/csscrush/CssCrush.php';
114 
115  //opzioni per il processore dei css
116  $settings= array(
117  'minify' => true,
118  'output_dir' => ROOT . '/themes/' . $PARAMETERS['themes']['current_theme'],
119  'versioning' => true,
120  'formatter' => 'block'
121  );
122  //imposta le opzioni per il processore dei css
123  csscrush_set('options',$settings);
124 
125  //INCLUSIONE DI UN AUTOLOADER STANDARD QUALORA QUALCUNO VOLESSE CIMENTARSI NELL'USO DELLA PROGRAMMAZIONE AD OGGETTI
126  /// [autoloader_example]
127  //include l'autoloader
128  require_once(ROOT . '/system/lib/dlight/core/autoloader/Autoloader.php');
129  //istanzia l'autoloader
130  $autoloader = new \dlight\core\autoloader\Autoloader();
131 
132  //aggiunge i path in cui cercare le risorse
133  $autoloader->addPath(ROOT . '/system/lib/');
134 
135  //aggiunge nell'array degli autori che sfruttano la classe phpbrowscap
136  $autoloader->addVendor('gdrcd');
137  $autoloader->addVendor('erusev');
138 
139  /// [autoloader_example]
140 
141  //INCLUSIONE DI UNA CLASSE CONTENITORE UTILE PER L'INIEZIONE DELLE DIPENDENZE SE QUALCUNO VOLESSE CIMENTARSI NELL'USO
142  //DELLA PROGRAMMAZIONE AD OGGETTI
143  //avvia la classe contenitore
144  \gdrcd\core\gdrcd::getInstance();
145  //imposta un alias più comodo per il contenitore delle classi
146  $gdrcd = \gdrcd\core\gdrcd::$class;
147 ```
148 Questo blocco di codice va aggiunto alla fine del codice prima del tag di chiusura di php.
149 La seconda procedura è indicata anche se si tenta di installate la patch su una versione del gdrcd diversa dal 5.5.

151 ## 4 Funzionalità introdotte {#mainpage_4} #
152 La patch introduce diversi strumenti rispetto alla vecchia versione del GDRCD, che seppur non visibili subito ad occhio
153 nudo forniscono delle utilità molto comode per lo sviluppo modulare del pacchetto anche in vista di una nuova versione
154 o lo sviluppo di moduli nuovi.
155 La patch contiene:
156 - Una integrazione con il preprocessore di css [csscrush](https://the-echoplex.net/csscrush/) pre avere una gestione dei
157 css più organizzata. Attualmente i css del gdrcd non sono stati modificati ma si può trovare nella cartella
158 **themes/advanced/css/source/gdrcd.css** e nei vari file presenti in quella sezione i file per iniziare a lavorare con il
159 preprocessore.
160 - Un nuovo gestore delle connessioni e operazioni con il database
161 - Dei set di funzioni per il caricamento delle risorse (funzioni,monuli,template) per facilizzare la gestione del lavoro.
162 I vari set di sunzioni hann l aloro guida specifica nelle varie pagine di questa documentaizone.
